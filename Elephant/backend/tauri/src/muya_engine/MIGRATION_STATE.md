# Migration state

Rust foundation established from Muya JavaScript commit `f482fa928effa4ad6bec4bc6f0d80d04ef57315d`.

## Architecture

The migration now has two Rust layers:

- `Elephant/crates/muya-core`: future DOM-independent editor core;
- `Elephant/backend/tauri/src/muya_engine`: temporary atomic JavaScript parity staging area.

Tauri consumes `muya-core` as a path dependency. The frontend still runs Muya JavaScript.

## Executable `muya-core` parser slice

Block parsing and Markdown serialization currently cover:

- ATX and Setext headings;
- paragraphs and thematic breaks;
- consecutive blockquotes;
- fenced code blocks;
- unordered, ordered and task lists;
- aligned GFM tables.

Inline parsing and serialization currently cover:

- escapes and code spans;
- images and inline links;
- strong, emphasis and strikethrough;
- soft and hard breaks;
- text fallback.

The parser builds structural list, item, row and cell nodes rather than flattening those constructions into strings. Inline nodes are recursive in normal text containers, while fenced code is kept literal.

This executable slice has Rust unit and round-trip tests, but full JavaScript-vs-Rust characterization is still required before runtime activation or JavaScript deletion.

Each atomic module has Rust unit tests. Deterministic modules are additionally compared against fixtures generated by executing the JavaScript oracle. Observable JavaScript details such as UTF-16 lengths and offsets, truthiness, stringification, insertion order, escaped table pipes and exact HTML output are represented explicitly.

No JavaScript oracle has been deleted yet.
