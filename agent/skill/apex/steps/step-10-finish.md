# APEX Step 10 — Finish

Summarize what changed and what was proven.

## Procedure

1. List changed files by category.
2. Mark each acceptance criterion as proven, not proven, or blocked.
3. Include exact validation commands and outcomes.
4. Mention skipped verification only with a concrete reason.
5. If a pull request is requested, keep the PR body factual and test-focused.

## Output

```md
## APEX complete

Task: ...
Criteria:
- [x] ...
Files changed:
- ...
Validation:
- ...
Verification:
- ...
Remaining risks:
- ...
```

## Rules

- Do not overclaim.
- Do not say a feature is fully working unless the relevant runtime or workflow was exercised.
- Keep the summary short enough to audit.
